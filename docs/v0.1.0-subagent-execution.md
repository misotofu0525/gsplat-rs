## Incremental Progress Update (2026-02-14)

This section tracks post-plan implementation progress after the initial `v0.1.0` execution snapshot.

### Completed Since Baseline

- SA-01 Core
  - `SceneBuffers` evolved to include `sh_degree` and flattened `sh_rest`.
  - Validation now enforces SH coefficient count consistency.
- SA-02 IO
  - PLY loader now supports ASCII, `binary_little_endian`, and `binary_big_endian`.
  - Added typed property parsing and contiguous `f_rest_0..N` extraction.
  - SH degree is inferred from `f_rest_*` count and surfaced in summaries.
  - Added parser tests for binary PLY + `f_rest_*`.
- SA-03/SA-05 Render Pipeline
  - Camera-to-view/projection application tightened in preprocess/render setup.
  - Added SH-based color evaluation (degree 0-4) from view direction.
  - WGSL fragment path now applies Gaussian-style radial falloff.
  - Added renderer readback path for CPU-side frame extraction.
  - Replaced isotropic size-based quads with covariance-driven anisotropic screen-space ellipse geometry.
  - Added Jacobian-projected covariance path (`J * V_cam * J^T`) and ellipse-axis raster support.
- SA-04 Sort
  - Backend naming aligned to odd-even terminology.
  - Added guard/fallback behavior for oversized GPU odd-even workloads.
- SA-05 Desktop Dev
  - CLI extended with `--frames`, `--width`, `--height`, `--orbit`, `--auto-camera`, `--png`.
  - Offscreen multi-frame rendering and PNG output implemented.
- SA-07 QA/Datasets
  - Added dataset fetch script: `tests/datasets/fetch-nvidia-flowers-1.sh`.
  - Verified real-scene render path with NVIDIA Flowers sample PLY.
- SA-08 Format
  - Packed format version bumped to v2 with SH metadata payload (`sh_degree`, `sh_rest`).
