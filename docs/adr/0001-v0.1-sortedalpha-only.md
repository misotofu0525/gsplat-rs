# ADR-0001: v0.1 Quality Contract is SortedAlpha Only

- Status: Accepted
- Date: 2026-02-12

## Context

The v0.1.0 delivery plan uses multiple rendering backends in different maturity levels.
Some backends (sort-free or research-oriented approximations) are useful for experiments but are not yet a stable quality baseline across platforms.

## Decision

For `v0.1.x`, only the `SortedAlpha` path is covered by the quality commitment:

- Conformance and release gates are validated against `SortedAlpha`.
- Experimental backends are allowed but explicitly out-of-contract.
- Any backend requiring matched training metadata is disabled by default.

## Consequences

- Release criteria remain measurable and deterministic.
- Experimental work can continue in non-blocking lanes without delaying `v0.1.0`.
- Documentation and APIs must keep this distinction explicit.
