# v0.1.0 多 Subagent 计划（Gate-Driven）

## 全局规则
1. 主线只承诺 `SortedAlpha` 质量路径。
2. 任何需要训练配套的方案不进入发布阻塞项。
3. 所有任务以“出口条件”判定完成，不以时间判定。
4. 每个 subagent 只改所属目录；跨目录变更需先过接口 Gate。

## Gate 流程

### G0 接口冻结
可开始：仓库骨架存在。  
并行执行：SA-01, SA-00。  
出口条件：
1. `gsplat-core` 公共类型冻结（相机、模式、统计、错误）。
2. C ABI 函数签名草案冻结。
3. ADR 记录“v0.1 仅承诺 SortedAlpha”。

阻塞条件：
1. 任何下游 crate 依赖的类型仍频繁变更。

### G1 数据输入闭环
可开始：G0 通过。  
并行执行：SA-02, SA-03(接口对齐), SA-07(测试样本准备)。  
出口条件：
1. PLY 必需字段解析通过。
2. 非法/缺失字段有明确错误码。
3. 可产出统一场景缓冲（供 preprocess 消费）。

阻塞条件：
1. 解析结果在不同平台行为不一致。

### G2 参考渲染闭环（发布主线）
可开始：G1 通过。  
并行执行：SA-03, SA-04, SA-05, SA-07。  
出口条件：
1. 可见集 + key 生成稳定。
2. 排序后端可运行（GPU 或 CPU fallback）。
3. SortedAlpha 可渲染稳定输出。
4. 帧统计可读：`frame/preprocess/sort/raster/visible/drawn`。
5. conformance baseline 生成并通过。

阻塞条件：
1. 只能在单平台跑通。
2. 统计缺失导致无法定位瓶颈。

### G3 平台接入闭环
可开始：G2 通过。  
并行执行：SA-06, SA-07。  
出口条件：
1. C ABI 可创建上下文、加载场景、渲染一帧、取 stats。
2. iOS demo 最小链路跑通。
3. Android demo 最小链路跑通。

阻塞条件：
1. FFI 生命周期不稳定（崩溃/泄漏）。

### G4 发布门禁
可开始：G3 通过。  
并行执行：SA-07, SA-08, SA-00。  
出口条件（全部满足才打 tag）：
1. `cargo check --workspace` 通过。
2. conformance 全通过（仅 SortedAlpha 作为质量承诺）。
3. 长稳运行通过（无明显内存增长）。
4. README/API/兼容性声明完成。
5. 发布说明明确：实验后端不纳入质量承诺。

阻塞条件：
1. 任一阻塞缺陷（崩溃、错误渲染、ABI 不稳定）未关闭。

## 非阻塞（Research Lane）
1. SortFree/WSR/Stochastic/Hybrid 等方案放入实验 lane。
2. 标记 `requires_matched_training` 的方案默认关闭。
3. 不影响 G4 打版。
