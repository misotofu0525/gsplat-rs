# Task Plans

- `active/`: task bundles for work currently in progress.
- `completed/`: historical task bundles and verification notes from completed
  work.

Completed bundles are kept for work history, but their path references should
stay aligned with the current repository layout. Current repository facts live
in `README.md`, `AGENTS.md`, and `handbook/`.

Each completed bundle should keep enough evidence to distinguish real completion
from an abandoned draft:

- `Status`: completed, superseded, or intentionally closed
- `Changed paths`: the main files or directories affected
- `Verification evidence`: commands that were run and their observed result
- `Known gaps`: remaining manual, device, browser, or release-bar checks
